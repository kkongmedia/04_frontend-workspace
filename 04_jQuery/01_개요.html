jQurey 개요

* jQurey란?

기존에 자바스크립으 언어만으로 구현하기엔 복잡했던 구문 작성들을
간소화를 하기 위해서 "존 레식"에 의해 만들어짐
Javascript 기반 라이브러리(유용한 기능들의 모음집) => ex) ojdbc6.jar 같은거...

복잡한 자바스크립트 구문을 보다 짧고 쉽게 개발 가능
(적은 양의 코드로 빠르고, 풍부한 기능 제공) >> jQurey 장점
1) DOM요소 => html 태그들.. 와 관련된 스크립트를 보다 쉽게 구현할 수 있음
2) Ajax(비동기식 통신), 이벤트 처리 등등 폭 넓게 지원함
3) jQurey와 관련된 확장된 플러그인, 오픈 api 등을 지원(차트, 슬라이드, 등등..)

* jQurey 라이브러리 연결 방법

head 태그 내에 script 태그로 외부 jQurey 관련 .js 파일 연결하기
단, 오프라인/ 온라인 방식이 있다. - jQurey 공식 사이트 : 공식사이트

1. jQurey 라이브러리 직접 다운로드 받은 후 경로 지정(오프라인) 2. CDN(Content Delievery Network) 방식
파일을 다운로드 받아서 저장할 필요없이 라이브러리를 제공하는 사이트 url 제시해서 파일경로 지정 (단, 인터넷이 연결되어 있어야 한다.)

* jQurey 구문 작성하기

jQurey 구문은 dom 요소를 다루는 구문을 기본적으로 작성할 것이기 때문에
문서상에서 요소들이 다 만들어지고 나서 실행되어야 한다.

> 순수 자바스크립트 에서는 

                window.onload = function(){
                    실행내용;   
                }

                Javascript 에서의 window.onload 구문은
                해당의 문서 요소들이 다 로드되고(그려지고, 렌더링되고) + 
                외부문서들(CSS 파일)도 다 가지고 온 뒤에 실행됨
                그리고 window.onload 구문은 해당 이 문서에서 단 한번만 작성 가능
                (여러개 있다면, 마지막에 작성된것 반영)
> jQurey 에서의 해당 내용은 3가지 구문으로 제공 
                1) jQurey(document).ready(function(){
                    실험내용;
                });

                2) $(document).ready(function(){
                    실험내용;
                })

                3) $(function(){
                    실험내용;
                })

                해당 문서의 요소들만 다 로드되면(그려지면, 렌더링되면) 곧바로 실행됨
                여러번 기술 가능함(제한없음) -> 순서대로 실행됨
